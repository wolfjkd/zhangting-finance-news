## pywebview 无边框模式

> **相关项目**：zhangting-finance-news（涨停财经聚合播报）

### 问题背景

用户希望窗口标题栏跟随暗色主题变化。最开始尝试使用 `frameless=True` 无边框模式，完全自定义标题栏。

### 问题现象

1. **窗口无法调整大小**：鼠标拖动边框只是平移窗口，无法改变宽高
2. **自定义按钮不生效**：最小化、最大化、关闭按钮点击无响应
3. **resize handle 被截获**：拖动边框时被 `-webkit-app-region: drag` 捕获，变成拖动窗口

### 根因分析

pywebview 的无边框模式存在以下问题：
- resize 逻辑依赖底层 GUI 框架实现，在 Windows 上行为不稳定
- `-webkit-app-region: drag` 会截获所有鼠标事件，导致 resize handle 无法接收事件
- 虽然可以尝试用 JS 手动计算 resize，但需要获取窗口句柄（HWND），pywebview 对此支持不完整

### 失败方案（勿重复）

```python
# 方案1：frameless + JS resize（失败）
window = webview.create_window(..., frameless=True)

# JS 中尝试 resize
pywebview.api.resize_window(newWidth, newHeight)

# 问题：resize 不生效，窗口大小不变
```

```javascript
// 方案2：mousedown 事件被 drag 截获（失败）
document.addEventListener('mousedown', onResize);
// 问题：即使加了 -webkit-app-region: no-drag，resize 仍不生效
```

### 成功方案：原生标题栏 + DWM API

**核心思路**：放弃无边框，使用原生标题栏保留所有窗口功能，通过 Windows DWM API 设置标题栏颜色。

**步骤1**：恢复原生标题栏
```python
window = webview.create_window(
    '窗口标题',
    url=html_path,
    frameless=False  # 使用原生标题栏
)
```

**步骤2**：导入 Windows API
```python
import ctypes
from ctypes import wintypes
```

**步骤3**：实现 DWM 标题栏颜色设置
```python
def set_titlebar_dark_mode(hwnd, dark=True):
    """通过 Windows DWM API 设置标题栏暗色模式"""
    try:
        dwmapi = ctypes.windll.dwmapi
        DWMWA_USE_IMMERSIVE_DARK_MODE = 20
        value = ctypes.c_int(1 if dark else 0)
        result = dwmapi.DwmSetWindowAttribute(
            hwnd,
            DWMWA_USE_IMMERSIVE_DARK_MODE,
            ctypes.byref(value),
            ctypes.sizeof(value)
        )
        return result == 0
    except Exception as e:
        logger.warning(f'设置标题栏暗色模式失败: {e}')
        return False
```

**步骤4**：前端调用 API
```python
def set_theme(self, theme):
    """设置主题（影响Windows标题栏颜色）"""
    try:
        import platform
        if platform.system() != 'Windows':
            return json.dumps({'status': 'unsupported'})

        hwnd = self._window.gui.renderer_hwnd
        if not hwnd:
            hwnd = ctypes.windll.user32.FindWindowW(None, '窗口标题')

        if hwnd:
            dark = (theme == 'dark')
            success = set_titlebar_dark_mode(hwnd, dark)
            return json.dumps({'status': 'ok' if success else 'failed'})
        return json.dumps({'status': 'no_hwnd'})
    except Exception as e:
        return json.dumps({'status': 'error', 'message': str(e)})
```

**步骤5**：前端切换主题时调用
```javascript
function applyTheme() {
    if (settings.darkMode) {
        document.body.classList.add('dark-mode');
    } else {
        document.body.classList.remove('dark-mode');
    }
    // 调用后端 API 设置系统标题栏颜色
    if (window.pywebview && window.pywebview.api) {
        pywebview.api.set_theme(settings.darkMode ? 'dark' : 'light').catch(() => {});
    }
}
```

### 方案对比

| 方案 | 窗口调整 | 最小化/最大化/关闭 | 标题栏跟随主题 | 稳定性 |
|------|---------|-------------------|---------------|--------|
| frameless + JS | ❌ 不生效 | ❌ 不生效 | ✅ 可以 | ❌ 差 |
| 原生 + DWM API | ✅ 正常 | ✅ 正常 | ✅ 可以 | ✅ 稳定 |

### 经验总结

1. **pywebview frameless 模式的 resize 能力有限**，在 Windows 上依赖底层实现，不可靠
2. **自定义标题栏按钮需要 -webkit-app-region: no-drag**，但这只解决事件问题，不解决 resize 问题
3. **Windows DWM API 可以直接控制原生标题栏颜色**，无需使用 frameless
4. **优先使用原生控件**，只有在原生控件无法满足需求时才考虑自定义

## Windows 标题栏颜色

> 参考：§pywebview 无边框模式

### DWM API 简介

Windows Desktop Window Manager (DWM) 提供了 `DwmSetWindowAttribute` 函数，可以设置窗口的渲染属性，包括标题栏的暗色模式。

### 关键常量

| 常量名 | 值 | 说明 |
|--------|---|------|
| DWMWA_USE_IMMERSIVE_DARK_MODE | 20 | 使用沉浸式暗色模式（Win10 1903+） |
| DWMWA_USE_IMMERSIVE_DARK_MODE | 19 | 旧版暗色模式常量（兼容） |
| DWMWA_CAPTION_COLOR | 35 | 标题栏颜色（需要 Win11） |

### 注意事项

1. **需要窗口句柄（HWND）**：可以通过 `pywebview` 的 `window.gui.renderer_hwnd` 获取，或使用 `FindWindowW`
2. **Win10/Win11 兼容性**：暗色模式在 Win10 1903+ 可用，更早版本可能无效
3. **系统主题影响**：如果用户关闭了系统级暗色模式，设置可能不生效

## pywebview 标题栏暗色模式（已验证）

> **相关项目**：zhangting-finance-news（涨停财经聚合播报）
> **验证日期**：2026-06-28
> **状态**：✅ 已验证可用

### 问题背景

用户希望 pywebview 桌面应用的 Windows 原生标题栏跟随暗色主题变化。

### 踩坑记录（失败方案，勿重复）

**坑1：renderer_hwnd 不是主窗口句柄**

```python
# ❌ 错误：renderer_hwnd 是浏览器控件的句柄，不是主窗口
hwnd = self._window.gui.renderer_hwnd
dwmapi.DwmSetWindowAttribute(hwnd, 20, ...)  # 不生效！
```

`renderer_hwnd` 是 EdgeChromium 浏览器控件的句柄，DWM API 需要的是**顶层主窗口句柄**。

**坑2：FindWindowW 按标题查找不可靠**

```python
# ❌ 不可靠：标题可能被修改、编码问题、多窗口混淆
hwnd = ctypes.windll.user32.FindWindowW(None, '窗口标题')
```

**坑3：JS localStorage 对 Python 不可见**

```python
# ❌ 错误：Python 启动时读不到 JS 的 localStorage
dark_mode = window.evaluate_js("localStorage.getItem('darkMode')")  # 可能返回 None
```

JS 的 `localStorage` 存在浏览器进程中，Python 端启动时 `evaluate_js` 可能还未就绪。

**坑4：pywebview API 时序问题**

```javascript
// ❌ 错误：初始化时 pywebview.api 可能还没准备好
applyTheme();  // 此时 window.pywebview 可能是 undefined
```

### 成功方案（7步完整实现）

**核心思路**：
1. 用 `GetAncestor(renderer_hwnd, GA_ROOT)` 从浏览器控件找到主窗口
2. DWM API 同时尝试常量 20 和 19（兼容不同 Windows 版本）
3. 设置通过 `persist_settings` API 同步到文件（Python 可读）
4. JS 端监听 `pywebviewready` 事件确保 API 就绪

**步骤1**：实现 DWM API（兼容双常量）
```python
import ctypes
from ctypes import wintypes

GA_ROOT = 2

def set_titlebar_dark_mode(hwnd, dark=True):
    """通过 Windows DWM API 设置标题栏暗色模式"""
    try:
        dwmapi = ctypes.windll.dwmapi
        # Win10 1903+ 用 20，旧版用 19
        for attr_id in [20, 19]:
            value = ctypes.c_int(1 if dark else 0)
            result = dwmapi.DwmSetWindowAttribute(
                hwnd, attr_id, ctypes.byref(value), ctypes.sizeof(value)
            )
            if result == 0:
                return True
        return False
    except Exception as e:
        return False
```

**步骤2**：实现 GetAncestor 获取主窗口句柄
```python
def _get_main_hwnd_from_renderer(renderer_hwnd):
    """从浏览器控件句柄获取顶层主窗口句柄"""
    try:
        main_hwnd = ctypes.windll.user32.GetAncestor(renderer_hwnd, GA_ROOT)
        return main_hwnd
    except Exception:
        return None
```

**步骤3**：实现带缓存的 HWND 查找方法（Api 类方法）
```python
def _find_hwnd(self):
    """查找并缓存主窗口句柄"""
    if self._main_hwnd:
        if ctypes.windll.user32.IsWindow(self._main_hwnd):
            return self._main_hwnd
        self._main_hwnd = None

    # 方法1: 从 renderer_hwnd 向上找
    try:
        renderer_hwnd = self._window().gui.renderer_hwnd
        if renderer_hwnd:
            main_hwnd = _get_main_hwnd_from_renderer(renderer_hwnd)
            if main_hwnd:
                self._main_hwnd = main_hwnd
                return main_hwnd
    except Exception:
        pass

    # 方法2: FindWindowW（备用）
    hwnd = ctypes.windll.user32.FindWindowW(None, '窗口标题')
    if hwnd:
        self._main_hwnd = hwnd
        return hwnd
    return None
```

**步骤4**：实现 persist_settings API（设置持久化到文件）
```python
def persist_settings(self, settings_json):
    """前端调用：保存设置到文件（供 Python 端读取）"""
    config_dir = os.path.join(os.environ.get('APPDATA', ''), 'AppName')
    os.makedirs(config_dir, exist_ok=True)
    config_path = os.path.join(config_dir, 'settings.json')
    with open(config_path, 'w', encoding='utf-8') as f:
        f.write(settings_json)
```

**步骤5**：启动时从文件 + JS 双保险读取设置
```python
def _apply_initial_theme(window):
    """启动时应用标题栏主题（双保险）"""
    import time
    time.sleep(1)
    dark_mode = False

    # 方法1: 从配置文件读取
    config_path = os.path.join(os.environ.get('APPDATA', ''), 'AppName', 'settings.json')
    if os.path.exists(config_path):
        with open(config_path, 'r', encoding='utf-8') as f:
            cfg = json.load(f)
            dark_mode = cfg.get('darkMode', False)

    # 方法2: 从 JS localStorage 读取（备用）
    if not dark_mode:
        result = window.evaluate_js(
            "(function() { try { return JSON.parse(localStorage.getItem('settings') || '{}').darkMode || false; } catch(e) { return false; } })()"
        )
        if result is True or str(result) == 'true':
            dark_mode = True

    # 查找主窗口句柄（双方法）
    hwnd = None
    try:
        renderer_hwnd = window.gui.renderer_hwnd
        if renderer_hwnd:
            hwnd = _get_main_hwnd_from_renderer(renderer_hwnd)
    except Exception:
        pass
    if not hwnd:
        hwnd = ctypes.windll.user32.FindWindowW(None, '窗口标题')

    if hwnd:
        set_titlebar_dark_mode(hwnd, dark_mode)
```

**步骤6**：JS 端 saveSettings 同步到 Python 文件 + pywebviewready 事件
```javascript
// saveSettings 同步到 Python 文件
function saveSettings() {
    try {
        localStorage.setItem('app-settings', JSON.stringify(settings));
        if (window.pywebview && window.pywebview.api && window.pywebview.api.persist_settings) {
            pywebview.api.persist_settings(JSON.stringify(settings)).catch(() => {});
        }
    } catch (e) {}
}

// 加载设置后立即保存到文件
let settings = loadSettings();
try { saveSettings(); } catch(e) {}

// pywebviewready 事件确保 API 就绪后应用主题
window.addEventListener('pywebviewready', () => {
    setWindowTitlebarTheme(settings.darkMode);
});

// 主题切换函数
function setWindowTitlebarTheme(isDark) {
    if (window.pywebview && window.pywebview.api && window.pywebview.api.set_theme) {
        pywebview.api.set_theme(isDark ? 'dark' : 'light').catch(() => {});
    }
}
```

**步骤7**：在 on_loaded 回调中启动初始主题应用
```python
def on_loaded_with_auth(window):
    threading.Thread(target=_apply_initial_theme, args=(window,), daemon=True).start()

window.events.loaded += on_loaded_with_auth
```

### 关键经验总结

1. **`renderer_hwnd` 不是主窗口句柄**：它是 EdgeChromium 浏览器控件的句柄，必须用 `GetAncestor(hwnd, GA_ROOT)` 找到顶层窗口
2. **DWM API 要试两个常量**：20（Win10 1903+）和 19（旧版），先试 20 再试 19
3. **JS localStorage 对 Python 不可见**：必须通过 `persist_settings` API 同步到文件
4. **pywebview API 有初始化时序**：JS 端必须监听 `pywebviewready` 事件，不能在脚本加载时直接调用
5. **HWND 要缓存**：用 `_find_hwnd()` 缓存句柄，避免每次查找
6. **双保险原则**：读设置用文件 + JS，找 HWND 用 GetAncestor + FindWindowW

### 方案对比

| 方案 | 窗口调整 | 标题栏跟随主题 | 稳定性 | 验证状态 |
|------|---------|---------------|--------|---------|
| frameless + JS | ❌ 不生效 | ✅ 可以 | ❌ 差 | ❌ 失败 |
| 原生 + renderer_hwnd 直传 | ✅ 正常 | ❌ 不生效 | ❌ 差 | ❌ 失败 |
| **原生 + GetAncestor + 设置持久化** | ✅ 正常 | ✅ 可以 | ✅ 稳定 | ✅ **已验证** |

## 配置文件存储规范

> **相关项目**：zhangting-finance-news（涨停财经聚合播报）
> **制定日期**：2026-06-30
> **状态**：✅ 已验证可用

### 问题背景

PyInstaller 打包单文件 exe 后，JS 的 localStorage 存储在浏览器进程的临时目录中，每次运行都会重新初始化，导致用户设置丢失。

### 存储位置

所有用户配置文件统一存放在 Windows 的 **AppData** 目录：

```
C:\Users\{用户名}\AppData\Roaming\ZTFINews\
```

### 存储文件清单

| 文件 | 用途 | 保存时机 |
|------|------|----------|
| `settings.json` | 用户设置（功能开关、关键词、主题色等） | 每次修改设置时 |
| `watchlist.json` | 自选股列表 | 添加/删除自选股时 |
| `watchlist_groups.json` | 自选股分组 | 修改分组时 |
| `stock_names.json` | 股票名称映射 | 搜索股票时 |
| `window_size.json` | 窗口尺寸 | 窗口调整结束后（1秒防抖） |
| `data_sources.json` | 数据源开关状态 | 切换数据源开关时 |
| `seen_aids.db` | 已读消息去重（SQLite） | 收到新消息时 |
| `app.log` | 日志文件 | 运行时持续写入 |
| `.state` | 时间回拨检测状态 | 程序关闭时 |

### 存储机制

**双重存储策略**：
1. **前端**：使用 `localStorage` 作为运行时缓存
2. **后端**：通过 Python 文件作为持久化存储

**同步流程**：
```
用户修改设置 → localStorage 更新 → 调用 pywebview.api.persist_settings() → Python 文件保存

程序启动 → pywebview.api.get_settings() → 读取文件 → 更新 localStorage → 应用设置
```

### 为什么放在 AppData 目录

| 优势 | 说明 |
|------|------|
| **用户无关** | 不依赖 exe 安装位置，用户移动 exe 不影响配置 |
| **系统标准** | 符合 Windows 应用配置存储规范 |
| **持久化** | 不会被杀毒软件误删，不会被系统清理 |
| **隔离性** | 每个用户独立配置，多人共用电脑互不影响 |
| **跨版本** | 版本升级后配置文件依然存在 |

### 关键代码

**后端保存设置**：
```python
def persist_settings(self, settings_json):
    settings = json.loads(settings_json)
    settings['config_version'] = '3.8.1'

    with open(config_path, 'w', encoding='utf-8') as f:
        f.write(json.dumps(settings))
    return json.dumps({'status': 'ok'})
```

## 配置版本管理机制

> **相关项目**：zhangting-finance-news（涨停财经聚合播报）
> **制定日期**：2026-06-30
> **状态**：✅ 已验证可用

### 问题背景

版本升级时，如果配置文件格式发生变化（新增字段、删除字段、重命名字段），直接加载旧配置可能导致程序崩溃或功能异常。

### 版本字段规范

| 配置文件 | 版本字段 | 当前版本 |
|----------|----------|----------|
| `settings.json` | `config_version` | 3.8.1 |
| `data_sources.json` | `version` | 3.8.1 |

### 版本升级策略

**启动时检测流程**：
```
程序启动 → 读取配置文件 → 检查版本号 → 对比当前版本 → 执行迁移 → 更新版本号 → 加载配置
```

**版本兼容性矩阵**：
